# PayPal 集成准备方案

## 现实检查

当前产品状态：
- ✅ 前端页面可用
- ✅ API 改写功能正常
- ❌ 没有用户系统（登录/注册）
- ❌ 没有数据库（无法记录谁付费了）
- ❌ 没有会话管理（无法识别"这是Pro用户"）
- ❌ 没有配额计数系统（500字/天限制）

**结论：现在集成 PayPal = 只收银子不发货（无法解锁功能）**

- 创建订阅计划
- 支付成功后标记用户为Pro
- 解锁无限字数+全部风格

## 但我现在给你准备好代码

下面是**Phase 4 的完整后端代码**，你可以先审阅，等 Phase 1-3 完成后再部署。

### 需要的环境变量

在 Render.com 添加：
```
PAYPAL_CLIENT_ID=你的PayPal Client ID
PAYPAL_CLIENT_SECRET=你的PayPal Secret
PAYPAL_MODE=sandbox  # 测试用，上线后改 live
```

### 获取 PayPal Client ID

1. 访问 https://developer.paypal.com
2. 登录你的 PayPal 账号
3. 进入 Dashboard → My Apps & Credentials
4. 创建 App，获取 Client ID 和 Secret

## 核心问题：没有用户系统，PayPal 怎么知道谁付了钱？

**方案 A：最简单（推荐先做）**
- 用户点击"Upgrade to Pro" → 弹邮箱输入框 → 输入邮箱 → PayPal 支付 → 后端记录"这个邮箱是Pro"
- 用户每次使用时输入邮箱 → 后端查数据库确认是否Pro
- 缺点：每次都要输邮箱，但 MVP 够用了

**方案 B：完整方案（等用户多了再做）**
- 注册账号（邮箱+密码）
- 登录后获得 session token
- 所有 API 请求带 token
- 后端验证 token + 查权限
